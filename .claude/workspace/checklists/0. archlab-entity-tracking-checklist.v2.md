## ARCHITECTURE INVARIANTS

These invariants apply to ALL phases and tasks:

1. **UUID is THE identity** - No component, element, session, or entity exists without a UUID
2. **EntityRegistry is THE source** - All entity lookup flows through EntityRegistry, no parallel ID systems
3. **No optional tracking** - Entity tracking is mandatory, not configurable
4. **No legacy IDs** - componentId, nodeId, poolId are REPLACED by entityId, not coexisting
5. **No escape hatches** - Every code path registers/unregisters, no bypasses
6. **Full lifecycle visibility** - Creation, modification, destruction all tracked and logged
7. **Full traceability** - Every entity supports error trace, leak detection, performance attribution, state history
8. **Complete persistence** - ALL app state (UI positions, settings, events, everything) persisted to DB
